4.4 Errors 41-50

E41: ERROR_CANT_USE_TYPE_IN_MODULO

E42: ERROR_CANT_USE_TYPE_IN_BITWISE_EXPRESSION

When trying to use a floating pointer number in any of these, this error will be thrown.

E43: ERROR_CANT_USE_TYPE_IN_NEGATION

E44: ERROR_CANT_USE_TYPE_IN_LOGICAL_EXPRESSION

E45: ERROR_IDENTIFIER_NOT_FOUND

Unused.

E46: ERROR_IDENTIFIER_INVALID

When the identifier in a DO call chain is invalid, this error will be thrown.

E47: ERROR_UNDEFINED_VARIABLE

When a variable is used before it is defined, this error will be thrown.

E48: ERROR_TOO_MANY_ARGUMENTS

When a standard library function is called with too many arguments, this error will be thrown.
Check chapter 3 for a list of standard library functions.

E49: ERROR_INVALID_CHAR

When writing a char literal, if the char is not valid, this error will be thrown.
Example: 

DO SAY ('Hello!'); // Invalid, a char literal must be one character long

E50: ERROR_INVALID_NUMBER

When writing a number literal, if the number is not valid, this error will be thrown.
Example: 

DO SAY (1F); // Invalid, a float literal must have a decimal point

RANGE and RANGEF functions also throw this error if you give 0 as a step, to prevent infinite loops
