4.2 Errors 21-30

E21: ERROR_EXPECTED_EXTENSION

The compiler expected an extension, but none was found.
Example: 

DO SAY ("Hello!") // if theres no semicolon, it expects an extension
MAKE INT x = 123; // this line is not an extension, so an error is thrown

E22: ERROR_EXPECTED_BLOCK

The compiler expected a block, but none was found.
Example: 

MAKE FUNC VOID myFunc (); // Error: A function must have a block.

E23: ERROR_EXPECTED_EXPRESSION

The compiler expected an expression, but none was found.
Things like WHILE, WHEN, IF, FOR, etc. expect an expression. right after
Example: 

DO SAYLN ("loop") WHILE; // WHILE expects an expression, an error is thrown

E24: ERROR_INVALID_EXPRESSION

The compiler found an invalid expression.
This is trown rarely, but it can happen.
It's usually pretty self-explanatory whats wrong if it happens.

E25: ERROR_OPERATOR_NOT_UNARY

The compiler expected a unary operator (-, !, ~, ^/ or !-), but found a binary operator.
Example: 

DO SAYLN (+2); // + is a binary operator, so it can't be used like a unary operator

E26: ERROR_INVALID_FUNCTION_DECLARATION_ARGUMENT

The compiler found an invalid function declaration argument.
Example: 

MAKE FUNC VOID myFunc (STRING name, age) {DO SAY ("func!");}; // Error: age doesn't have it's type set

E27: ERROR_UNEXPECTED_TOKEN

Unused

E28: ERROR_PROCESS_NOT_RUNNING

If this error is thrown, report it to the developer.
This error is thrown when the compiler tries to access the process, but it's not running.
The process should always be running when the compiler tries to access it.

E29: ERROR_INTERPRETER_INVALID_COMMAND

If this error is thrown, report it to the developer.
This error is thrown when the interpreter tries to execute an invalid command.
The interpreter should never try to execute an invalid command.

E30: ERROR_SAY_ERROR

This error is thrown if the SAY function fails to print the message.
