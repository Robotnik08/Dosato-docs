4.0 Errors 1-10

An error is a problem that occurs when the program is running.
There are two types of errors: syntax errors and runtime errors.
Syntax errors are caught before the code is ran, these can not be catched by the user.
Runtime errors are caught while the code is running, these can be catched by the user.
And uncaught error will cause the program to safely exit.
The exit code then is set to the error code.
In most cases a helpful error message is printed to the console, also showing where the error occured.
When no error occured, the program will always exit with exit code 0.

If the program crashes without printing an error message, it is most likely a bug and should be reported.
If the error location is wrong or the error message is not correct, it is most likely a bug and should be reported.

E1: ERROR_SYNTAX

This is a general error, this is currently not used, in favour of more detailed errors.

E2: ERROR_TYPE_MISMATCH

This error occurs when a variable is used in an expression or system function with a type that is not compatible with the variable.
A common example: 

MAKE ARRAY INT numbers = [1, 2, 3];
SET numbers = 1; // Error, numbers can not be converted to array.

E3: ERROR_UNDEFINED

E4: ERROR_RECURSION

E5: ERROR_MEMORY

E6: ERROR_FILE

E7: ERROR_RUNTIME

E8: ERROR_PARSER

These are currently not used.
These can be freely used by the user.

E9: ERROR_EXPECTED_MASTER

This is a parser error, if the first token of a line is not a master token (DO|SET|MAKE), throw this error.
A common example: 

DO SAY ("Hello world!"); // Correct
SAY ("Hello world!"); // Error, expected master token (DO).

E10: ERROR_EXPECTED_IDENTIFIER

This can be a parser error and a runtime error, if the token is not an identifier while it's expected to be, throw this error.
A common example: 

MAKE INT num1 = 321; // Correct
MAKE INT = 123; // Error, expected identifier.
