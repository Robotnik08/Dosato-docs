4.5 Errors 51-60

E51: ERROR_CAST_ERROR

The given value cannot be casted to the given type.
Because of the autocasting, theres not a lot of cases where this error can occur.

E52: ERROR_CANNOT_MODIFY_CONSTANT

The given value is a constant and cannot be modified.
Right now thats only certain standard library constants.

E53: ERROR_CANNOT_MODIFY_LITERAL

E54: ERROR_EXPECTED_REFRENCE

Unused

E55: ERROR_INVALID_REFRENCE_EXPRESSION

When trying to access a refrence, the given operator is not correct (expecting # operator).

E56: ERROR_ARRAY_OUT_OF_BOUNDS

When trying to access an array with an index that is out of bounds, this error will be thrown.
Example: 

MAKE ARRAY INT arr = [1, 2, 3];
DO SAYLN (arr[3]); // arr[3] is out of bounds, so an error is thrown

E57: ERROR_EXPECTED_ARRAY

when making an array with MAKE, the compiler expects an array, but none was found.
or, a for loop expects an array, but none was found.
Examples: 

MAKE ARRAY INT arr = 123; // Error: expected array, found int
DO SAY (i) FOR (123 => i); // Error: expected array, found int

E58: ERROR_INCORRECT_ARRAY_DEPTH

When casting two arrays, if the dimensions do no match, this error will be thrown.

E59: ERROR_INVALID_CAST

When a typecast has an invalid type:
(ARRAY INT) is valid, but (INT ARRAY) is not.

E60: ERROR_ARRAY_CAST_ERROR

When an array can't cast it's type, this error will be thrown.
