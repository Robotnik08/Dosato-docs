4.1 Errors 11-20

E11: ERROR_WRONG_BRACKET_ROUND

The compiler expected round brackets, but got different brackets instead.
Example: 

DO SAY ["Hello World"]; // Error, Expected round bracket.

E12: ERROR_WRONG_BRACKET_SQUARE

E13: ERROR_WRONG_BRACKET_CURLY

Currently are not in use, they are here for consistency.

E14: ERROR_EXPECTED_ARGUMENTS

The compiler found a function call/creation without argument brackets.
Example: 

DO SAY; // Error, Expected arguments.
MAKE FUNC VOID sayHello; // Error, Expected arguments.

E15: ERROR_EXPECTED_ARGUMENT

The compiler wanted an argument, but didn't find one.
Example: 

DO SAY ("Hello",); // Because of the comma, the compiler expects another argument.

E16: ERROR_EXPECTED_TYPE

The compiler expected a type (INT, STRING, etc...), but didn't find one.
Example: 

MAKE number = 5; // Error, Expected type.

E17: ERROR_EXPECTED_ASSIGN_OPERATOR

The compiler expected an assignment operator, but didn't find one.
Example: 

MAKE INT number 5; // Error, Expected assignment operator.

E18: ERROR_EXPECTED_COMMA

Unused.

E19: ERROR_EXPECTED_SEPERATOR

The compiler expected a seperator (semicolon), but didn't find one.
This happens mainly after a MAKE or SET, because a DO can accept extensions.
Example: 

SET number = 12 // Error, expected a seperator.

E20: ERROR_EMPTY_BLOCK

A function can't be empty, it needs at least one line.
A work around is just to return instantly.
Example: 

MAKE FUNC VOID doNothing() {
    // Error, empty block.
};
